# 28장: TypeScript SDK 실행 어댑터 — `runStreamed()`는 어떻게 실제 Codex CLI가 되는가

> **이 장의 질문**: Desktop bridge server가 `Codex.startThread().runStreamed()`를 호출하면 실제로 어떤 프로세스가 실행되고, 어떤 이벤트가 돌아오는가?

## 학습 목표

이 장에서는 Desktop이 사용하는 TypeScript SDK를 실행 어댑터로 읽습니다.

- `Codex` 클래스가 `CodexExec`를 감싸는 구조를 이해한다.
- `Thread.runStreamed()`가 `codex exec --experimental-json` stdout line을 `ThreadEvent`로 파싱하는 과정을 본다.
- `thread.started`, `turn.started`, `item.*`, `turn.completed`, `turn.failed` 이벤트가 Desktop Evidence의 원천이라는 점을 확인한다.
- SDK가 hidden chain-of-thought가 아니라 외부 event stream만 다룬다는 경계를 이해한다.

## 1단계: `Codex`는 thread factory다

`sdk/typescript/src/codex.ts`의 `Codex` 클래스는 직접 turn을 실행하지 않습니다. 생성자에서 `CodexExec`를 만들고, `startThread()`로 `Thread`를 반환합니다.

```ts
export class Codex {
  private exec: CodexExec;

  constructor(options: CodexOptions = {}) {
    const { codexPathOverride, env, config } = options;
    this.exec = new CodexExec(codexPathOverride, env, config);
  }

  startThread(options: ThreadOptions = {}): Thread {
    return new Thread(this.exec, this.options, options);
  }
}
```

Desktop bridge server는 이 factory를 사용해 workspace별 thread를 만듭니다.

## 2단계: `Thread.runStreamed()`는 AsyncGenerator를 반환한다

`sdk/typescript/src/thread.ts`에서 `runStreamed()`는 즉시 전체 결과를 반환하지 않고 event generator를 돌려줍니다.

```ts
async runStreamed(input: Input, turnOptions: TurnOptions = {}): Promise<StreamedTurn> {
  return { events: this.runStreamedInternal(input, turnOptions) };
}
```

내부에서는 입력을 prompt와 image 배열로 정규화한 뒤 `this._exec.run(...)`을 호출합니다. stdout line을 하나씩 JSON parse해서 `ThreadEvent`로 yield합니다.

```ts
for await (const item of generator) {
  const parsed = JSON.parse(item) as ThreadEvent;
  if (parsed.type === "thread.started") {
    this._id = parsed.thread_id;
  }
  yield parsed;
}
```

이 구조 때문에 Desktop은 turn이 끝날 때까지 기다리지 않고 event가 도착할 때마다 UI와 evidence archive를 갱신할 수 있습니다.

## 3단계: 실제 실행은 `codex exec --experimental-json`

`sdk/typescript/src/exec.ts`의 `CodexExec.run()`은 command args를 조립합니다.

```ts
const commandArgs: string[] = ["exec", "--experimental-json"];
```

그리고 thread option을 CLI flag/config로 바꿉니다.

| SDK option | CLI/config |
| --- | --- |
| `model` | `--model` |
| `sandboxMode` | `--sandbox` |
| `workingDirectory` | `--cd` |
| `skipGitRepoCheck` | `--skip-git-repo-check` |
| `modelReasoningEffort` | `--config model_reasoning_effort="..."` |
| `approvalPolicy` | `--config approval_policy="..."` |
| `threadId` | `resume <thread-id>` |

마지막으로 child process stdin에 prompt를 쓰고, stdout line을 readline으로 읽습니다.

```ts
child.stdin.write(args.input);
child.stdin.end();

const rl = readline.createInterface({
  input: child.stdout,
  crlfDelay: Infinity,
});
```

즉 Desktop이 "Codex 앱을 실행한다"는 말은 Electron이 마법처럼 내부 API를 호출한다는 뜻이 아닙니다. TypeScript SDK가 `codex exec --experimental-json` 프로세스를 띄우고 JSONL stdout을 SDK event로 파싱한다는 뜻입니다.

## 4단계: 이벤트 타입은 SDK 계약이다

`sdk/typescript/src/events.ts`는 Desktop이 받아야 하는 top-level event를 정의합니다.

| event | 의미 |
| --- | --- |
| `thread.started` | 첫 turn에서 thread id가 확정됨 |
| `turn.started` | 사용자 prompt가 모델 turn으로 들어감 |
| `item.started` | 새 item이 시작됨 |
| `item.updated` | 실행 중 item 상태가 갱신됨 |
| `item.completed` | item이 완료됨 |
| `turn.completed` | turn이 usage와 함께 완료됨 |
| `turn.failed` | turn 실패 |
| `error` | stream fatal error |

Desktop evidence normalizer는 이 이벤트들을 source event로 삼습니다. tool name만 보지 않고 `item` 내부 payload, status, result, usage를 함께 봐야 하는 이유가 여기에 있습니다.

## Claim -> file path -> observable event/check

| Claim | file path | observable event/check |
| --- | --- | --- |
| SDK의 entry class는 thread factory다 | `sdk/typescript/src/codex.ts` | `Codex.startThread()`가 `new Thread(...)`를 반환한다 |
| streaming turn은 AsyncGenerator다 | `sdk/typescript/src/thread.ts` | `runStreamed()`가 `{ events }`를 반환하고 내부 generator가 `yield parsed`를 수행한다 |
| 실제 실행은 Codex CLI child process다 | `sdk/typescript/src/exec.ts` | `spawn(this.executablePath, ["exec", "--experimental-json", ...])`를 호출한다 |
| prompt는 stdin으로 들어간다 | `sdk/typescript/src/exec.ts` | `child.stdin.write(args.input)` 후 `child.stdin.end()`를 호출한다 |
| SDK event type은 명시 union이다 | `sdk/typescript/src/events.ts` | `ThreadEvent` union에 `thread.started`, `turn.completed`, `item.completed` 등이 있다 |

## 강좌 체크포인트
