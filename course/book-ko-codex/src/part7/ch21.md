# 21장: TUI 표면 — `/goal` 입력은 어떻게 앱 이벤트가 되는가

> **이 장의 질문**: 사용자가 `/goal`을 입력했을 때, Codex TUI는 무엇을 사용자 메시지로 보내고 무엇을 control plane 이벤트로 처리하는가?

## 학습 목표

이 장에서는 `/goal`의 첫 관문인 TUI를 읽습니다. 목표는 세 가지입니다.

- `/goal`이 command palette에 어떻게 등록되는지 확인한다.
- `/goal <objective>`와 `/goal pause|resume|clear`가 어디서 분기되는지 따라간다.
- TUI가 모델에게 텍스트를 보내지 않고 app-server RPC로 상태를 바꾸는 지점을 이해한다.

## 1단계: slash command 등록

`codex-rs/tui/src/slash_command.rs`에서 `/goal`은 `SlashCommand::Goal` variant입니다. 설명은 "set or view the goal for a long-running task"입니다. 같은 파일에서 두 가지 중요한 속성이 보입니다.

```rust
SlashCommand::Goal => "set or view the goal for a long-running task",
```

```rust
SlashCommand::Goal
```

`supports_inline_args()`에 포함되기 때문에 `/goal improve tests`처럼 command 뒤의 텍스트를 objective로 받을 수 있습니다. `available_during_task()`에도 포함되어 있어 이미 작업이 도는 중에도 goal 상태를 조회하거나 pause/resume/clear할 수 있습니다.

## 2단계: bare `/goal`

bare `/goal`은 objective가 없습니다. `chatwidget/slash_dispatch.rs`의 `dispatch_command`는 먼저 `Feature::Goals`가 켜져 있는지 봅니다. thread id가 있으면 `OpenThreadGoalMenu` 이벤트를 보냅니다. thread가 아직 없으면 사용법을 출력합니다.

```rust
SlashCommand::Goal => {
    if !self.config.features.enabled(Feature::Goals) {
        return;
    }
    if let Some(thread_id) = self.thread_id {
        self.app_event_tx
            .send(AppEvent::OpenThreadGoalMenu { thread_id });
    } else {
        self.add_info_message(
            GOAL_USAGE.to_string(),
            Some(GOAL_USAGE_HINT.to_string()),
        );
    }
}
```

여기서 중요한 점은 bare `/goal`이 모델에게 "현재 goal을 알려줘"라고 묻지 않는다는 것입니다. TUI가 app event를 만들고, app layer가 app-server의 `thread/goal/get`을 호출합니다.

## 3단계: `/goal pause`, `/goal resume`, `/goal clear`

inline args가 있는 경우 `dispatch_prepared_command_with_args`로 들어갑니다. 여기서 먼저 `pause`, `resume`, `clear`를 control command로 인식합니다.

```rust
let control_command = match trimmed.to_ascii_lowercase().as_str() {
    "clear" => Some(GoalControlCommand::Clear),
    "pause" => Some(GoalControlCommand::SetStatus(AppThreadGoalStatus::Paused)),
    "resume" => Some(GoalControlCommand::SetStatus(AppThreadGoalStatus::Active)),
    _ => None,
};
```

control command라면 TUI는 app event를 보냅니다.

```rust
GoalControlCommand::Clear => {
    self.app_event_tx
        .send(AppEvent::ClearThreadGoal { thread_id });
}
GoalControlCommand::SetStatus(status) => {
    self.app_event_tx
        .send(AppEvent::SetThreadGoalStatus { thread_id, status });
}
```

이것이 `/goal` 표면에서 가장 중요한 분리입니다. 사용자가 상태를 바꾸는 명령은 모델의 natural language turn이 아니라, deterministic app event입니다.

## 4단계: `/goal <objective>`

control command가 아니면 나머지 inline args는 objective입니다. TUI는 먼저 빈 objective와 길이 제한을 검사합니다. thread id가 아직 없으면 이 입력을 queue에 넣고, session이 시작된 뒤 slash command로 다시 파싱되도록 합니다.

```rust
self.queue_user_message_with_options(
    UserMessage {
        text: format!("/goal {args}"),
        local_images: Vec::new(),
        remote_image_urls: Vec::new(),
        text_elements: Vec::new(),
        mention_bindings: Vec::new(),
    },
    QueuedInputAction::ParseSlash,
);
```

thread id가 있으면 `SetThreadGoalObjective` 이벤트를 보냅니다. mode는 `ConfirmIfExists`입니다. 이미 목표가 있다면 바로 덮어쓰지 않기 위한 장치입니다.

```rust
self.app_event_tx.send(AppEvent::SetThreadGoalObjective {
    thread_id,
    objective: objective.to_string(),
    mode: ThreadGoalSetMode::ConfirmIfExists,
});
```

## 5단계: app layer action

`codex-rs/tui/src/app/thread_goal_actions.rs`는 이 이벤트를 app-server RPC로 바꿉니다.

- `open_thread_goal_menu` -> `thread_goal_get`
- `set_thread_goal_objective` -> `thread_goal_set(objective, Active, None)`
- `set_thread_goal_status` -> `thread_goal_set(None, status, None)`
- `clear_thread_goal` -> `thread_goal_clear`

목표 교체 확인도 여기서 처리됩니다. `ConfirmIfExists` 모드에서 먼저 `thread_goal_get`을 호출하고, goal이 있으면 "Replace goal?" 선택 UI를 띄웁니다.

## UI 표시

bare `/goal`이 goal을 읽으면 `goal_menu.rs`가 history에 요약을 추가합니다.

```rust
Line::from("Goal".bold()),
Line::from(vec!["Status: ".dim(), goal_status_label(goal.status).to_string().into()]),
Line::from(vec!["Objective: ".dim(), goal.objective.clone().into()]),
```

footer 표시도 따로 있습니다. `goal_status.rs`는 active goal이면 token budget 또는 elapsed time을 compact string으로 만들고, `bottom_pane/footer.rs`는 이를 `Pursuing goal`, `Goal paused`, `Goal achieved` 같은 한 줄로 보여 줍니다.

## Claim -> file path -> observable event/check

| Claim | file path | observable event/check |
| --- | --- | --- |
| `/goal`은 inline argument를 받는 slash command다 | `codex-rs/tui/src/slash_command.rs` | `SlashCommand::Goal`이 `supports_inline_args()`에 포함된다 |
| bare `/goal`은 app event를 만든다 | `codex-rs/tui/src/chatwidget/slash_dispatch.rs` | thread id가 있으면 `OpenThreadGoalMenu`를 보낸다 |
| `pause/resume/clear`는 모델 입력이 아니다 | `codex-rs/tui/src/chatwidget/slash_dispatch.rs` | control command가 `SetThreadGoalStatus` 또는 `ClearThreadGoal`로 변환된다 |
| objective 설정은 기존 goal 확인을 거친다 | `codex-rs/tui/src/app/thread_goal_actions.rs` | `ConfirmIfExists`일 때 먼저 `thread_goal_get`을 호출한다 |
| goal 요약은 TUI history와 footer에 표시된다 | `codex-rs/tui/src/chatwidget/goal_menu.rs`, `codex-rs/tui/src/chatwidget/goal_status.rs` | status/objective/usage line과 `GoalStatusIndicator`가 있다 |
